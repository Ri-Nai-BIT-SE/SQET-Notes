# 白盒测试可视化演示

这些交互式组件可以帮助你直观理解白盒测试的核心概念。

## 1. 语句与分支覆盖演示

<CodeCoverageDemo />

**说明**：
- 左侧显示代码，右侧可以输入测试数据
- **绿色高亮**的行表示被执行的代码
- 尝试输入不同的年龄值（如 15 和 20），观察哪些代码行被执行
- 注意：输入 20 时，else 分支（第6行）不会执行；输入 15 时，if 内部（第4行）不会执行

**学习要点**：
- **语句覆盖**：所有代码行都至少执行一次
- **分支覆盖**：每个判断的真假分支都要走到
- 要达到 100% 分支覆盖，需要至少两个测试用例（一个走 if，一个走 else）

## 2. 路径覆盖演示（逻辑流程图）

<FlowChartDemo />

**说明**：
- 通过勾选"是会员"和"有库存"来改变输入条件
- **蓝色高亮**的路径表示当前执行的路径
- 观察不同条件组合下，程序走的是哪条路径

**学习要点**：
- **路径覆盖**：覆盖程序中所有可能的路径
- 这个例子有 3 条路径：
  - 路径1：会员 + 有库存 → 发货
  - 路径2：会员 + 无库存 → 缺货
  - 路径3：非会员 → 无权限
- 要达到 100% 路径覆盖，需要 3 个测试用例

## 3. 边界值分析演示

<BoundaryDemo />

**说明**：
- 输入不同的数值，观察数轴上的指针位置
- **黄色高亮**的边界点（1 和 10）是测试重点
- 当输入值等于边界值时，会显示特殊提示

**学习要点**：
- **边界值**：错误往往出现在边界上
- 标准边界值测试（5点法）：
  - 最小值 (1)
  - 略大于最小值 (2)
  - 正常值 (5)
  - 略小于最大值 (9)
  - 最大值 (10)
- **健壮性测试**（7点法）还要加上：0（min-）和 11（max+）

## 4. 基本路径测试综合案例（大题实战）

<ComplexityDemo />

**说明**：
- 这是一个完整的**基本路径测试**综合演示，对应期末考试中的**10分大题**
- 左侧显示 C 代码，可以调整变量 A、B、X 的值
- 右侧动态显示**控制流图 (CFG)**，实时高亮执行路径
- 观察代码行高亮与图形节点高亮的同步效果

**核心考点演示**：

1. **复合条件拆分**：
   - `if (A > 1 && B == 0)` 被拆分为节点 1（A>1）和节点 2（B==0）
   - `if (A == 2 || X > 1)` 被拆分为节点 4（A==2）和节点 5（X>1）
   - **考点**：画控制流图时，复合逻辑必须拆分为原子逻辑

2. **汇聚节点**：
   - 节点 4 是第一个 IF 语句结束后的汇聚点
   - **考点**：画图时容易漏掉汇聚节点，导致图形结构错误

3. **圈复杂度计算**：
   - 判定节点数 (P) = 4
   - 公式：**V(G) = P + 1 = 5**
   - 至少需要 **5条** 独立路径才能覆盖所有逻辑

4. **独立路径探索**：
   尝试以下测试用例，观察不同的执行路径：
   - `A=0, B=0, X=0` → Path: 1→4→5→7（短路逻辑演示）
   - `A=2, B=0, X=0` → Path: 1→2→3→4→6→7
   - `A=2, B=1, X=0` → Path: 1→2→4→6→7
   - `A=3, B=0, X=2` → Path: 1→2→3→4→5→6→7
   - `A=3, B=1, X=0` → Path: 1→2→4→5→7

**学习要点**：
- **动态路径追踪**：红色虚线动画直观展示代码执行流
- **代码-图形映射**：左侧代码行高亮与右侧图形节点同步
- **交互式学习**：通过改变输入值，理解什么是"独立路径"
- **大题准备**：这是期末考试最可能考到的题型

## 使用建议

1. **交互学习**：通过改变输入值，观察代码执行路径的变化
2. **理解概念**：直观看到"覆盖"的含义
3. **考试准备**：帮助理解各种覆盖标准的区别
4. **大题训练**：使用综合案例组件练习基本路径测试的完整流程
