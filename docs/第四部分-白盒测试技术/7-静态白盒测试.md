**出处**：PPT P102-103，`output_4_15_1.txt`（结尾课堂练习）

静态白盒测试是通过**代码审查**来发现错误，不需要运行程序。

## 课堂练习例题

**例题**：老师给了一段 `冒泡排序` 的代码（PPT P103），让大家找错。

## 找错方向（考点）

### 1. 规范性错误

- **缩进**：代码缩进不规范
- **注释**：缺少必要的注释或注释不清晰
- **命名**：变量、函数命名不规范（如使用拼音、缩写不清晰）

### 2. 逻辑错误

- **数组越界**：C语言常见，`a[10]` 访问到了 `a[10]`（应该是 `a[0]` 到 `a[9]`）
- **变量未初始化**：使用未初始化的变量
- **死循环**：循环条件错误导致无限循环
- **算法效率低**：使用了低效的算法实现

### 3. 安全性错误

- **内存泄漏**：动态分配的内存没有释放
- **空指针访问**：访问空指针
- **缓冲区溢出**：数组越界写入

## 常见错误示例

### 数组越界（C语言）

```c
int a[10];
for (int i = 0; i <= 10; i++) {  // 错误：应该是 i < 10
    a[i] = i;  // 当 i=10 时，访问 a[10] 越界
}
```

### 变量未初始化

```c
int sum;
for (int i = 0; i < 10; i++) {
    sum += i;  // 错误：sum 未初始化
}
```

### 死循环

```c
int i = 0;
while (i < 10) {
    // 错误：缺少 i++，导致死循环
    printf("%d\n", i);
}
```
