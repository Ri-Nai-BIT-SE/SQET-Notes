# 3. 老师亲授的"得分（避坑）"秘籍

### 🛑 关于大题的"新"判卷标准

老师提到这两年的出题风格变了：

*   **以前**：给你一张白纸，让你写设计过程，写得沾边就给辛苦分。
*   **现在**：**"挖空填空"式的大题**。
    *   题目会画好表格或图的大致框架，挖出很多空让你填（比如等价类编号、覆盖的路径序号、圈复杂度数值）。
    *   **风险**：**客观性极强，错了就是错了，没有辛苦分！**
    *   **对策**：计算圈复杂度 `V(G)` 时，一定要用三个公式互相验算，保证那个数字绝对正确！

### 🛑 关于答题规范

1.  **标识符必写**：设计测试用例时，表格里必须有一列叫 **"测试用例ID"** (TC_01, TC_02...)。老师原话："每年都有同学因为没写编号扣分"。
2.  **预期输出必写**：不要只写输入数据，必须写上程序应该返回什么结果。
3.  **不要用铅笔**：答题纸要归档，必须用黑色签字笔。
4.  **术语要准**：案例分析题里，如果让你分析性能，不要用大白话（比如"电脑卡了"），要用术语（"**响应时间**过长"、"**CPU利用率**达到瓶颈"、"**内存泄漏**"）。
