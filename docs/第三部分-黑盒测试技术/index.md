# 第三部分：黑盒测试技术（**期末大题重点**）

> **考试重要性**：这部分在期末考试中通常对应**一道10分左右的设计大题**（如：等价类划分+边界值，或因果图+判定表），并且在选择题中也会频繁出现。

## 黑盒测试概述

### 核心逻辑与依据

*   **别名**：功能测试、数据驱动测试。
*   **核心思想**：把程序看作一个**打不开的黑盒子**。测试人员完全不关心盒子内部的代码逻辑和结构，只关心**输入**和**输出**是否符合预期。
*   **依据文档**：**需求规格说明书（Requirement Specification）**。这是考试中画圈的考点，黑盒测试是基于需求文档设计的。
*   **目的**：**证伪**（找错）。重点检查功能遗漏、接口错误、数据结构错误、性能错误等。

### 特点

- 不关注程序内部结构
- 只关注输入和输出
- 基于需求规格说明

## 🎯 快速复习指南

### 必考方法（大题重点）

- **等价类划分法**：画表、编号、覆盖原则（有效类合并，无效类单独）
- **判定表法**：条件/动作、规则数计算（2^n）、简化规则
- **因果图法**：符号系统（关系符号、约束符号）、转换为判定表

### 选择题重点

- **边界值分析**：4n+1公式、5点法、7点法、最坏情况测试
- **覆盖原则**：有效类尽可能多覆盖，无效类每次只覆盖一个
- **符号识别**：因果图的关系符号和约束符号

### 实战练习

- **三角形问题**：等价类、边界值、判定表三种方法都要会
- **Pascal标识符**：完整的等价类划分实战演练
- **自动售货机**：因果图+判定表的经典案例
