# 测试用例设计规范

### 测试用例必须包含的要素

**缺一不可，否则扣分**：

1. **测试用例ID（标识）**：
   - 格式：TC_01, TC_02, ... 或 用例1, 用例2, ...
   - **老师原话**："每年都有同学因为没写编号扣分"
   - **【录音重点】**：缺少"编号"必扣分

2. **输入数据**：
   - 明确列出所有输入值
   - 对于三角形问题：A=3, B=4, C=5

3. **预期输出**：
   - **必须写**，不要只写输入数据
   - 明确说明程序应该返回什么结果
   - 例如：预期输出 = 一般三角形
   - **【录音重点】**：缺少"预期输出"必扣分

### 测试用例的定义与设计原则

**定义**：能够独立测试的最小单元，包含**输入**、**预期输出**（必须有，用于比对）、标识符。

**设计原则**：
- **经济性**：用最少的用例发现最多的缺陷（如等价类划分，避免重复测同一个区间）
- **设计出来的，不是写出来的**：要有组织性、逻辑性
- **黑盒测试关键点**：既要测正确的输入（功能实现），也要测**错误/异常**的输入（健壮性）
- **AI/大模型背景下的挑战**：传统测试预期结果是确定的（如1+1=2），大模型输出具有不确定性，预期结果可能是一个范围或模糊描述，这对测试执行带来新挑战

**规范要求**：
- **必须有**：标识符（ID）、测试项、环境、输入、**预期输出**
- **用例维护**：是持续改进的过程，要结合测试工具（如Excel或专门系统）进行跟踪（执行率、通过率）

### 简答/应用设计题得分关键点

**题目形式**：给一段自然语言描述的需求（如"大学生宿舍分配系统"或"登录页面"），要求设计测试用例。

**得分关键点**：
- **必须包含三列**：**ID (编号)**、**输入数据**、**预期输出**
- **缺少"预期输出"必扣分**
- **缺少"编号"必扣分**
- **覆盖度**：必须包含正常路径（Valid）和异常路径（Invalid/边界值）
- **足够的用例数**：题目会要求"给出足够的测试用例"，不要只写一两个

**完整示例**：
| 用例ID | 输入 (A, B, C) | 预期输出 |
|--------|---------------|---------|
| TC_01 | (3, 4, 5) | 一般三角形 |
| TC_02 | (3, 3, 4) | 等腰三角形 |

## 📋 测试用例设计的一般步骤

### 标准流程

1. **测试需求分析**：
   - 理解需求规格说明书
   - 识别输入和输出
   - 确定测试范围

2. **输入输出分析**：
   - 列出所有输入条件
   - 列出所有可能的输出结果
   - 识别隐含条件

3. **编写测试用例**：
   - 选择合适的测试方法（等价类、边界值、判定表等）
   - 设计测试用例
   - 确保用例包含：ID、输入、预期输出

4. **评审测试用例**：
   - 检查用例的完整性
   - 检查用例的有效性
   - 确保覆盖充分

5. **跟踪测试用例**：
   - 执行测试用例
   - 记录测试结果
   - 跟踪缺陷

## 🎯 复习建议

结合详细的PPT内容，复习重点应调整为：

### 1. 边界值分析法

- **熟练掌握 4n+1 公式**：n个变量的标准边界值测试
- **区分三种测试类型**：
  - 标准边界值测试（5点法）：4n+1 个用例
  - 健壮性测试（7点法）：增加无效边界
  - 最坏情况测试（笛卡尔积）：k^n 个用例
- **理解单故障假设**：标准边界值测试基于此假设

### 2. 判定表法

- **能够看懂并绘制判定表**：
  - 理解组成部分（条件桩、动作桩、条件项、动作项）
  - 掌握建立步骤
  - 理解规则数计算：2^n（n个条件）
- **掌握简化规则**：
  - 动作相同且只有一个条件不同 → 可以合并
  - 用"—"表示无关条件
- **理解三角形问题的判定表构建过程**：
  - 识别条件（a=b?, a=c?, b=c?, 构成三角形条件）
  - 识别动作（等边、等腰、一般、非三角形、不可能）
  - 排除"不可能"的规则

### 3. 因果图法

- **识记因果图的符号**（必考）：
  - **关系符号**：恒等(=)、非(~)、或(∨)、与(∧)
  - **约束符号**：E(互斥)、I(包含)、O(唯一)、R(要求)、M(屏蔽)
- **理解符号含义**：
  - 考试可能会给出图形让你识别含义
  - 或者给一段逻辑描述让你选择对应的因果图符号
- **掌握转换过程**：因果图 → 判定表 → 测试用例

### 4. 等价类划分法

- **掌握覆盖原则**：
  - 有效等价类：尽可能多覆盖（合并测试）
  - 无效等价类：每次只覆盖一个（避免错误屏蔽）
- **理解"潜规则"**：
  - 区间的划分（必须拆分为两个无效类）
  - 集合的划分
  - 布尔值的划分

### 5. 综合应用

- **三角形问题**：经典例题，必须熟练掌握
  - 等价类划分视角
  - 边界值分析
  - 判定表方法
- **Pascal标识符**：完整的实战演练
- **NextDate问题**：日期计算的复杂逻辑
- **ATM机/自动售货机**：状态转换和因果图

### 6. 测试用例设计规范

- **必须包含三要素**：ID、输入、预期输出
- **缺一不可，否则扣分**
- **格式要规范**：使用标准表格格式
