# 错误猜测法（Error Guessing）

### 核心思想

依靠经验和直觉，列举出所有可能的错误和容易发生错误的特殊情况。

### 具体做法

1. **列举可能的错误**：
   - 基于测试人员的经验和直觉
   - 考虑容易出错的特殊情况

2. **经验来源**：
   - 以前版本的问题
   - 由语言/OS/浏览器限制带来的问题
   - 模块间关联问题
   - 修复错误引入的新错误

3. **建立检查表（Checklist）**：
   - 积累常见错误
   - 形成错误检查清单
   - 便于后续测试参考

### 特点

- **优点**：能发现一些通过系统化方法难以发现的错误
- **缺点**：依赖个人经验，不够系统化
- **适用**：作为其他测试方法的补充

### 常见错误类型

1. **输入错误**：
   - 空输入
   - 特殊字符
   - 超长输入
   - 非法格式

2. **边界错误**：
   - 0值
   - 负值
   - 最大值+1
   - 最小值-1

3. **状态错误**：
   - 未初始化
   - 已删除
   - 并发访问

4. **逻辑错误**：
   - 除零
   - 空指针
   - 数组越界
