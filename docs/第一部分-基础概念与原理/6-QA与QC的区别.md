# QA（质量保证）与 QC（质量控制）的区别

### 基本区别

| 项目 | QA（质量保证） | QC（质量控制） |
|------|---------------|---------------|
| **关注点** | 过程（Process） | 产品（Product） |
| **方法** | 审计、预防、检查和改进 | 检测、发现问题 |
| **目标** | 预防缺陷 | 发现缺陷 |
| **性质** | 管理性工作 | 技术性工作 |
| **对象** | 流程和过程 | 过程和产物（产品） |
| **时机** | 贯穿整个开发过程 | 主要在开发完成后 |

### 详细说明

**QA（质量保证）**：
- **定义**：一种有计划的、系统化的行动模式，为项目或产品符合已有技术需求提供充分的信任
- **工作方式**：通过预防、检查和改进来保证质量
- **采用方法**：全面质量管理和过程改进管理
- **关注**：软件质量的检查和测量，包括度量的选取
- **审查对象**：整个开发流程（包括开发流程、测试流程等）

**QC（质量控制/测试）**：
- **工作方式**：执行软件，运行程序，找出产品中的缺陷
- **关注**：对过程的产物（产品）进行检查，而不是过程本身
- **活动**：对开发文档和源代码进行审查，运行软件以找出问题

### 经典比喻：餐厅的例子

**测试员（QC）**：
- 相当于餐厅的"试菜员"
- 检查菜品的口味、辣度是否符合川菜馆的标准
- 关注的是**产品**（菜品）本身

**质量保证人员（QA）**：
- 相当于餐厅的"流程审计员"
- 审查整个餐厅的流程：餐厅经理的流程、服务员的流程、试菜员的流程、厨师的流程
- 关注的是**流程**本身

### 实际工作中的情况

**注意**：在实际企业中，很多部门虽然叫"QA部门"，但可能：
- 只做QC的工作（测试）
- 同时做QA和QC的工作
- 真正只做QA的工作（流程审计）

**判断方法**：需要查看组织的质量管理体系文件，看其职责划分。

**记忆要点**：
- **QA**：关注过程，审计，预防，管理性工作
- **QC**：关注产品，检测，发现问题，技术性工作
