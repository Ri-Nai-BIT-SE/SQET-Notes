# 文档测试 (Documentation Testing)

### 重要性

文档是软件的一部分（程序+数据+文档）。

### 测试范围

*   **用户手册**：用户使用指南
*   **安装指南**：软件安装说明
*   **联机帮助**：在线帮助系统
*   **开发文档**：API文档、技术文档等

### 检查点（考点）

*   **准确性**：描述是否准确，与软件功能是否一致
*   **时效性**：截图是否为最新版本（防止滞后性）
*   **一致性**：术语是否一致，文档与软件是否一致
*   **完整性**：内容是否完整，没有遗漏
*   **易理解性**：是否容易理解，语言是否清晰

### 测试方法

*   主要通过**静态审查 (Review)** 进行
*   检查一致性、完整性、易理解性、准确性、时效性

### 考试注意

*   判断题：文档测试主要检查一致性、准确性、易理解性
*   选择题：文档测试的检查点包括准确性、时效性、一致性
*   作为项目实践的必选项
