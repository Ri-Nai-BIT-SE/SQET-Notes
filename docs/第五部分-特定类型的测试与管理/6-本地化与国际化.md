# 本地化与国际化

> **考试定位**：选择/判断题考点

### 🌍 核心概念

#### 国际化 (I18N - Internationalization)

*   **定义**：是基础。设计时使代码能处理多语言（如支持 Unicode、分离资源文件）
*   **阶段**：属于 Engineering 阶段
*   **目的**：为本地化做准备

**设计阶段考虑**：
- 多语言支持
- 字符编码（UTF-8）
- 文本方向（LTR/RTL）
- 文化适应性
- 分离资源文件（文本、图片等）

#### 本地化 (L10N - Localization)

*   **定义**：是过程。将产品加工以满足特定市场（语言、文化）
*   **关系**：**国际化是本地化的前提**

### 🚩 测试重点（不仅仅是翻译！）

#### 1. 语言翻译

*   界面文本翻译
*   帮助文档翻译
*   错误提示翻译

#### 2. 度量衡

*   英制/公制转换
*   斤/公斤、英里/公里等

#### 3. 时区与日期格式

*   时区设置
*   日期格式（YYYY-MM-DD vs MM/DD/YYYY）
*   时间格式（12小时制 vs 24小时制）

#### 4. 货币格式

*   货币符号（¥、$、€等）
*   货币单位
*   汇率显示

#### 5. 文化传统（重点）

*   **文本方向**：如阿拉伯语从右向左（RTL）
*   **宗教禁忌**：某些颜色、数字、图案的禁忌
*   **颜色喜好**：不同文化对颜色的理解不同
*   **键盘布局**：不同语言的键盘布局

#### 6. 文本扩展 (Text Expansion)

*   **问题**：翻译后文本变长导致 UI 溢出或截断
*   **示例**：英文转德文通常变长
*   **测试**：需要检查界面布局是否适配

#### 7. 功能性

*   本地化后不能破坏原有功能
*   快捷键冲突
*   热键设置
*   功能按钮位置

### 考试注意

*   **判断题**：
    *   "软件本地化仅仅是翻译语言" → **错误**（还包括度量衡、时区、文化等）
    *   "国际化是本地化的基础和前提" → **正确**
*   **选择题**：
    *   本地化测试包括哪些内容（翻译、度量衡、时区、文化等）
    *   文本扩展问题（UI溢出）
