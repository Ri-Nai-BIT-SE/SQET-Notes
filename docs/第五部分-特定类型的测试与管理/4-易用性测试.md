# 易用性测试 (Usability Testing)

### 核心

易理解、易学习、易操作、吸引用户。

### 🚩 考点（判断题）

**"不要让易用性的主观性妨碍测试工作"** → **正确**

即虽然易用性主观，但仍需客观标准。

### 测试内容

1. **安装测试**：安装过程是否简单明了
2. **功能易用性**：功能是否容易找到和使用
3. **界面测试 (UI)**：
   - **规范性**：符合设计规范
   - **一致性**：快捷键、布局、风格保持一致
4. **辅助系统测试**（**重点**）：
   - **帮助文档**：必须有，且内容准确
   - **向导**：操作引导，帮助用户完成任务
   - **信息提示**：必须有，且提示准确

### 考试注意

*   判断题：易用性虽然主观，但不能因此不进行测试
*   选择题：辅助系统包括帮助文档、向导、信息提示
*   作为项目实践的必选项
