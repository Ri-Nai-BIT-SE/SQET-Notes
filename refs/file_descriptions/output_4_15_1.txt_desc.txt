4.  **Logical Coverage Analysis (逻辑覆盖分析):**
    *   This is the core of the lecture and a major exam subject. The professor systematically explains different levels of test coverage, using a sample C program to illustrate each concept.
    *   The types of coverage discussed, in order of increasing strength, are:
        *   **Statement Coverage (语句覆盖):** Ensures every line of code is executed at least once. It is presented as the weakest form of coverage.
        *   **Decision Coverage (判定覆盖 / Branch Coverage):** Ensures every branch (e.g., both the `if` and `else` parts of a conditional) is taken.
        *   **Condition Coverage (条件覆盖):** Ensures each boolean sub-expression within a decision evaluates to both true and false. The lecture clarifies that this does not guarantee full Decision Coverage.
        *   **Decision/Condition Coverage (判定/条件覆盖):** A combination of the two, ensuring all branches are taken and all sub-expressions are evaluated to both true and false.
        *   **Multiple Condition Coverage (条件组合覆盖):** A more rigorous method that tests every possible combination of outcomes for the conditions within each decision.
        *   **Path Coverage (路径覆盖):** The most thorough method, aiming to test every possible execution path through the program. The professor notes this is often impractical or impossible for complex programs with loops.

5.  **Basic Path Testing (基本路径测试):**
    *   As a practical alternative to full path coverage, Basic Path Testing is introduced.
    *   **Control Flow Graph (控制流图):** The first step is to convert the program's logic into a graphical representation. The lecture explains the rules for creating this graph.
    *   **Cyclomatic Complexity (圈复杂度):** A key metric calculated from the control flow graph. It quantifies the program's complexity and determines the minimum number of independent paths that need to be tested to ensure full coverage. The professor provides three formulas for its calculation and emphasizes its importance for the exam.
    *   The goal is to design a set of test cases that covers this minimal set of independent paths, providing a thorough yet manageable testing strategy.
